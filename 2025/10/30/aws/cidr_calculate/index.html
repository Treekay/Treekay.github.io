CIDR calculate example
2025-10-30 16:28   label AWS

Questions

Company W has a VPC with an IPv4 CIDR block of 10.0.0.0/16, as shown in the figure below. The company is international, with multiple branch locations. The company has 3 branches in each city. The company operates in 13 cities in the north hemisphere and 45 cities in the south hemisphere. According to the customer distributions, the company decided all the branches in the north hemisphere will have 3 subnets at each location and all the branches in the south hemisphere will have 2 subnets at each location.

  1. What is the appropriate CIDR block size for this network address allocation scheme?
  2. What are the first and last usable host addresses in the 27th subnet (NOT AWS specific)?
  3. According to the figure, if Company N would

Resolving Steps

  1. Calculate the block size by the number of subnets needed
  2. Calculate the subnet network id
  3. the first usable address = subnet network address + 1
    the last usable address = broadcast address - 1 (broadcast address = network address + nums of addresses in each subnet - 1)

Answer

  1. 13 * 3 * 3 + 45 * 3 * 2 = 387,
    28 = 256 < 387,
    29 = 512 > 387,
    so block size = 16 + 9 = 25
  2. each subnets has 27 = 128 ip,
    for the 27th subnet, 26 * 128 = 3328, 3328/256 = 13, 3328 mod 256 = 0,
    so the 27th subnet is: 10.0.13.0/25
    the first usable host address: 10.0.13.0 + 1 = 10.0.13.1
    the last usable host address: 10.0.13.127 − 1 = 10.0.13.126
  3. internet gateway attach to VPC, supporting connecting VPC to the internet; for public subnets route table, target 0.0.0.0/0 to igw;
    for instances, allocate ip address for each instance to support communicate with the internet.
    Ensure security group associated with the ec2 instance and acl associated with the public subnet allowing inbound and outbound traffic.
