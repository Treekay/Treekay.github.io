Automation Design

题型方向 常见关键词 典型 AWS 服务
弹性扩展 (Scaling) auto scale / high demand / variable workload Auto Scaling Group, ELB
自动部署 (Deployment) manage resources / consistent setup / repeatable process CloudFormation, Elastic Beanstalk
监控与恢复 (Monitoring & Recovery) detect failures / ensure availability CloudWatch, Lambda, SNS
安全与运维自动化 (Ops Automation) patch / configuration / compliance Systems Manager, Config, Inspector
成本优化自动化 shut down idle / schedule start-stop Lambda, EventBridge, CloudWatch

Template

Step 1 — 说明要自动化的内容（What）

Identify which part of the cloud environment needs automation e.g., scaling, deployment, monitoring, patching, or cost control.

Step 2 — 指定具体 AWS 服务（How）

Name the service and briefly describe how it works. e.g., “Use AWS Auto Scaling to automatically adjust EC2 capacity based on CPU usage.”

Step 3 — 连接 Well-Architected Framework 原则（Why）

Tie it to a pillar:

  • Operational Excellence: automate deployments (CloudFormation)
  • Reliability: automated recovery and failover (CloudWatch + Lambda)
  • Performance Efficiency: auto scaling to meet demand
  • Cost Optimization: automatically remove idle resources

场景和服务

场景 AWS 服务 Well-Architected Pillar 说明
自动伸缩 Auto Scaling + ELB Performance Efficiency 应对需求波动，维持性能
自动部署 CloudFormation / Elastic Beanstalk Operational Excellence 模板化、可重复部署
自动恢复 CloudWatch Alarm + Lambda Reliability 自动重启或替换异常实例
自动补丁 Systems Manager Patch Manager Operational Excellence 减少人工维护错误
自动监控报警 CloudWatch + SNS Reliability 即时通知系统状态
成本调度 Lambda + EventBridge Cost Optimization 定时关闭空闲实例
配置合规监控 AWS Config + CloudTrail Security / Governance 自动检测配置变更
