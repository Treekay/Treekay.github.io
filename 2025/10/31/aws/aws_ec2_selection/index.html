AWS EC2 selection

识别工作负载类型

  • 长期稳定运行 → predictable workload
  • 临时任务 / 测试 → short-term / flexible
  • 计算密集 → CPU/GPU heavy
  • 不常使用 → low utilization

选择合适的定价模型 (Pricing Model)

模型 适用场景 成本特点 举例句式
On-Demand 临时测试、开发环境 成本最高、灵活 “Use On-Demand for flexibility.”
Reserved Instances / Savings Plans 长期稳定运行任务 成本低（最高可省 72%） “Use Reserved Instances for predictable workloads.”
Spot Instances 容错任务、批量分析 成本最低，但可能中断 “Use Spot Instances for cost saving in flexible jobs.”
Dedicated / Host Instances 合规、安全要求高 成本高 “Use Dedicated for compliance isolation.”

选择合适的实例类型 (Instance Type)

类型 特点 适用场景
General (t, m) 平衡 CPU/内存 Web server, app server
Compute optimized (c) 高 CPU 性能 Batch computing, video encoding
Memory optimized (r, x) 大内存 Database, caching
Storage optimized (i, d) 高 IO 性能 Data warehouse, Hadoop
GPU (g) 图形渲染 / AI 训练 Graphic design, ML training

附加优化策略

Auto Scaling → 动态扩展

Elastic Load Balancing → 负载分配

EBS-backed instances → 停止后数据保留

Custom AMI → 保持配置一致

AWS Systems Manager / CloudWatch → 自动打补丁、监控

Example template

Company X should match each workload with the right EC2 pricing model. Use Reserved Instances for long-running tasks, Spot Instances for variable or flexible workloads, and On-Demand for short-term development. Enable Auto Scaling to handle demand changes.

Use GPU-based instance because designers need high-performance graphics for rendering images and videos.

Use EBS-backed EC2 instances, which allow users to stop and start instances without losing data. Designers can pause work and resume later from the same state.

Create a custom Amazon Machine Image (AMI) from the configured instance. This ensures every new instance launched has the same software and settings for consistent collaboration.

Key factors: cost-effective, compliant, flexibility, performance, and reliability, secure, long-term data protection.

EC2 相关核心服务对比总结表

功能 服务/选项 说明
定价模型 On-Demand / Reserved / Spot / Dedicated 成本与灵活度权衡
数据持久化 EBS-backed vs Instance Store EBS 保存数据，Instance Store 随实例删除
配置保持 AMI (Amazon Machine Image) 保存系统设置与软件环境
自动扩展 Auto Scaling Group (ASG) 按需增加/减少实例
负载均衡 Elastic Load Balancer (ELB) 分配流量防止过载
维护与监控 Systems Manager / CloudWatch 自动补丁、性能监控

常见题型总结

题型 解题关键词 常用方案
成本优化类 “cost-effective / optimize / reduce cost” Reserved + Spot + Auto Scaling
高性能计算类 “graphics / video / simulation” GPU instance (G5/G4dn)
弹性伸缩类 “traffic spikes / unpredictable workload” Auto Scaling + On-Demand
定期任务类 “periodic / batch job” Spot + Lambda
配置保持类 “same configuration / resume work” EBS-backed + AMI
灾备/高可用类 “redundancy / backup” Multi-AZ + Load Balancer
