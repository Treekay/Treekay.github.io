AWS Decoupling Severless and Microservices
2025-10-31 00:05   label AWS

Decoupling and serverless, Microservices

题型 核心目标 常用 AWS 服务 答题关键词 常见陷阱
Decoupling 减少组件间依赖，使系统更灵活、更易扩展 SQS, SNS, EventBridge, S3, API Gateway asynchronous, loose coupling, message queue 忘记提“message queue”或“asynchronous communication”
Serverless 去除服务器管理，按需执行代码、自动扩展 Lambda, API Gateway, S3, DynamoDB, CloudFront managed, pay-per-use, auto scale 忘记提“cold start”或“runtime limits”缺点
Microservices 将单体应用拆分为独立模块，各自部署维护 ECS/EKS, Fargate, API Gateway, CloudWatch, X-Ray containerized, isolated, independently deployable 忘记提“monitoring complexity”或“data consistency”挑战

(1) Decoupling 题型

Solution: Use Amazon SQS or SNS to decouple components, so that backend services process requests asynchronously. Benefits: Improves scalability, fault tolerance, and allows components to fail independently. Cons: Adds message delivery delay and complexity in tracking asynchronous processes.

(2) Serverless 题型

Solution: Use AWS Lambda for backend functions triggered via API Gateway, and host static frontend on S3 + CloudFront. Benefits: No server management, automatic scaling, cost-efficient for variable workloads. Cons: Cold start latency and limited execution duration.

(3) Microservices 题型

Solution: Break the monolithic app into independent microservices using ECS/EKS with API Gateway for routing. Benefits: Each service can scale and deploy independently; failures are isolated. Cons: Increases operational complexity and requires centralized logging and monitoring (CloudWatch).

Pillar 对应三类题重点
Performance Efficiency Auto scaling, serverless event-driven design
Reliability Decoupling via SQS/SNS, fault isolation in microservices
Operational Excellence Automation, observability (CloudWatch, X-Ray)
Cost Optimization Serverless and event-driven workloads
Security IAM roles for each microservice/function
