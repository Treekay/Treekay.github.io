AWS S3 selection

读题关键条件：

  • 访问频率（frequent / infrequent / rarely accessed）
  • 保存时间（e.g. 3 years / 10 years）
  • 访问速度要求（immediate / within minutes / within hours）
  • 成本限制（cost-effective solution）
  • 合规要求（must retain / cannot delete / audit needs）
  • 灾备要求（backup / multi-region / redundancy）

判断最合适的 S3 存储类型（或组合）

  • 高访问频率：S3 Standard
  • 不常访问但需要快速取回：S3 Standard-IA
  • 很少访问（几小时取回即可）：S3 Glacier Flexible Retrieval / Deep Archive
  • 动态变化：S3 Intelligent-Tiering

指出关键服务或机制：

  • Lifecycle policy（自动在不同 tier 间转移）
  • Versioning（保留文件历史）
  • Cross-region replication (CRR)（灾备）
  • S3 Object Lock（合规要求，不可修改/删除）
  • AWS Backup / AWS Glacier Vault Lock（长期合规与安全）

常见 S3 存储类型对比表

S3 Type Access Frequency Retrieval Speed Cost (Storage) Use Case Notes
S3 Standard Frequent Immediate 💰💰💰 Active data, websites, analytics High availability, 3 AZs
S3 Intelligent-Tiering Variable Immediate 💰💰 Data with unpredictable access Auto-moves to cheaper tier
S3 Standard-IA (Infrequent Access) Less frequent Immediate 💰 Backup, disaster recovery Retrieval cost per GB
S3 One Zone-IA Less frequent Immediate 💰 Secondary backup Stored in one AZ only
S3 Glacier Instant Retrieval Rarely Seconds–Minutes 💰 Archive data needing quick access Cheapest instant retrieval
S3 Glacier Flexible Retrieval Rarely Minutes–Hours 💵 Archive, compliance storage Retrieval within 1–5 hours
S3 Glacier Deep Archive Very rare Up to 12 hours 💵 (lowest) Long-term backup (≥7–10 years) Cheapest, for legal or compliance data

应用场景总结

Scenario Recommended Storage Key AWS Features
Data accessed daily S3 Standard High durability and availability
Data after 60 days, rarely accessed Lifecycle → Glacier Flexible Retrieval Cost-effective long-term
Legal records for 10 years Glacier Deep Archive + Object Lock Compliance and WORM protection
Disaster recovery copy Cross-Region Replication Multi-region redundancy
Data with unpredictable usage Intelligent-Tiering Automatic tiering and cost saving

Example template

Use Amazon S3 with a lifecycle policy. Store active data in S3 Standard, and move older or rarely accessed data to S3 Glacier (or Glacier Deep Archive) for cost optimization. For redundancy and disaster recovery, enable cross-region replication and versioning/Object Lock to protect data from loss or modification.
