What are the main components of SQL?

  • DDL (Data Definition Language)
  • DML (Data Manipulation Language)
  • Views
  • Transaction control
  • Authorization
  • Embedded SQL and dynamic SQL

What does numeric(p,d) mean in SQL?

numeric(p,d) stores a fixed-point number with total p digits and d digits after the decimal. Example: numeric(3,1) allows 44.5 but not 444.5 or 0.32.

What is the basic structure of an SQL query?

select A1, A2, …, An
from r1, r2, …, rm
where P

  • The result is a relation.
  • SQL is case-insensitive for keywords and identifiers (e.g., name ≡ NAME).

How does SQL handle duplicates in SELECT results?

  • By default, duplicates may appear.
  • Use DISTINCT to remove duplicates.
  • Use ALL to explicitly retain duplicates.

What can you do in the SELECT clause besides listing columns?

  • Select all columns with *
  • Do arithmetic (e.g., salary/12)
  • Rename output columns with AS

Use WHERE with logical and comparison operators.

Use LIKE with patterns:

What set operations does SQL support and how do they treat duplicates?

  • UNION, INTERSECT, EXCEPT remove duplicates by default.

How do NULLs behave in SQL and how do you test for them?

  • NULL means unknown or not applicable.
  • Any arithmetic with NULL yields NULL.
  • Comparisons with NULL are unknown (not true).
  • WHERE filters out rows where the predicate is unknown.
  • Test with IS NULL / IS NOT NULL.

How do aggregate queries work (GROUP BY and HAVING)?

  • Aggregates: avg, min, max, sum, count
  • SELECT may include aggregate functions and columns listed in GROUP BY only.
  • HAVING filters groups; WHERE filters rows before grouping.
  • Logical order: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY

What kinds of subqueries are there and where can they appear?

  • WHERE: IN/NOT IN, SOME, ALL, EXISTS/NOT EXISTS (supports correlated subqueries)
  • FROM: derived tables (subquery as a relation), WITH clause for temporary named results (CTEs)
  • SELECT: scalar subqueries that return a single value

How do you modify data and schemas in SQL?

  • INSERT: add rows (optionally listing columns or inserting from a subquery)
  • DELETE: remove rows (optionally with subqueries)
  • UPDATE: change values; can use WHERE, CASE, and scalar subqueries
  • Schema changes: DROP TABLE; ALTER TABLE ADD column D; ALTER TABLE DROP column (DB-dependent)

What does a NATURAL JOIN do in SQL?

It matches rows that agree on all common attribute names between two tables and returns only one copy of each common attribute (deduplicates shared columns).

Why can NATURAL JOIN be dangerous, and how do you avoid issues?

If unrelated columns share the same name, NATURAL JOIN will equate them incorrectly. Avoid this by specifying the match explicitly with JOIN … USING (col_list) or JOIN … ON condition.

When should you use JOIN … USING vs JOIN … ON?

Use USING when matching on identically named columns (e.g., USING (course_id)). Use ON for general predicates or when column names differ (e.g., ON a.id = b.student_id and a.year >= b.year).

What’s the difference between INNER JOIN and OUTER JOIN?

INNER JOIN returns only matching rows. OUTER JOIN returns matching rows plus non-matching rows from one or both sides, padding missing columns with NULLs.

How do LEFT, RIGHT, and FULL OUTER JOIN differ?

LEFT OUTER JOIN keeps all left-table rows; RIGHT OUTER JOIN keeps all right-table rows; FULL OUTER JOIN keeps all rows from both tables, filling unmatched sides with NULLs.

What is the difference between a join condition and a join type?

The join condition specifies how rows match (e.g., ON a.id = b.id). The join type (inner/left/right/full) specifies how to treat non-matching rows.

What is a View and why use it?

A view is a virtual relation defined by a query (CREATE VIEW v AS ). It provides abstraction/security by exposing only needed columns/rows; the data is generated on demand from base tables.

How do materialized views differ from regular views?

Materialized views store the query result physically for speed but require maintenance on base-table changes. Regular views store only the definition and compute results at query time.

When are view updates generally allowed?

Typically only on simple views over a single base table without aggregates, DISTINCT, GROUP BY/HAVING, or expressions; the SELECT list contains only base columns; omitted base columns must allow NULL/defaults.

What is a transaction and how does it end?

A transaction is an atomic sequence of operations (the “magic bubble”) isolated from others. It ends with COMMIT (make changes permanent) or ROLLBACK (undo all changes in the transaction).

Which single-table integrity constraints are common and what do they enforce?

NOT NULL (no missing values), UNIQUE (no duplicates), PRIMARY KEY (UNIQUE + NOT NULL), and CHECK (custom predicate per row, e.g., CHECK (semester IN (‘Fall’,‘Winter’,‘Spring’,‘Summer’))).

What is referential integrity and how can violations be handled?

A foreign key requires referenced values to exist in another table (usually that table’s primary key). On delete/update, you can RESTRICT/REJECT (default), CASCADE, SET NULL, or SET DEFAULT.

What advanced SQL data types are useful beyond basic strings/numbers?

Date/time types: DATE, TIME, TIMESTAMP, INTERVAL. Large objects: BLOB (binary), CLOB/TEXT (character), typically returned via locators/pointers. User-defined types/domains: CREATE TYPE/DOMAIN, often with constraints (e.g., NOT NULL, CHECK).

What is an index and how do you create one?

An index is a data structure that speeds equality and range searches. Create with: CREATE INDEX idx_name ON table_name (col_list); the optimizer uses it automatically when beneficial.

How is authorization managed in SQL (users, privileges, roles)?

GRANT/REVOKE manage privileges (SELECT, INSERT, UPDATE, DELETE; plus DDL like CREATE, DROP). WITH GRANT OPTION lets a user pass privileges on. Privileges on a view don’t imply privileges on base tables. Roles group privileges and can be granted to users and other roles.
