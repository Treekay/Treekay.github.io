Brief summary for SQL

  • SQL overview: evolving standards (SQL-86 to SQL:2003); key components include DDL, DML, Views, Transaction control, Authorization, Embedded/Dynamic SQL.

  • Data Definition (DDL): CREATE TABLE with types (char, varchar, numeric(p,d)); integrity constraints (PRIMARY KEY, FOREIGN KEY, NOT NULL); composite keys and referential integrity.

  • Basic queries: SELECT–FROM–WHERE; case-insensitive; duplicates allowed by default; use DISTINCT or ALL; SELECT *; expressions and column renaming with AS; joins via FROM + WHERE; self-joins with table aliases.

  • Filtering: logical operators (AND/OR/NOT); comparisons (<, <=, >, >=, =, <>); BETWEEN; tuple comparisons.

  • Strings: LIKE with % (substring) and _ (single char); ESCAPE for literals; functions such as concatenation (||), UPPER/LOWER, LENGTH, SUBSTRING.

  • Ordering: ORDER BY with ASC/DESC; multiple sort keys supported.

  • Set operations: UNION, INTERSECT, EXCEPT (and their ALL variants); default behavior removes duplicates unless ALL is used.

  • Null semantics: NULL means unknown/absent; arithmetic/comparisons with NULL yield unknown; three-valued logic in WHERE; test with IS NULL / IS NOT NULL.

  • Aggregates: AVG, MIN, MAX, SUM, COUNT; GROUP BY forms groups; HAVING filters groups (vs WHERE filters rows); typical evaluation order: FROM → WHERE → GROUP BY → HAVING → SELECT.

  • Subqueries:

    • In WHERE: IN/NOT IN, SOME/ALL, EXISTS/NOT EXISTS, UNIQUE; supports correlated subqueries.
    • In FROM: derived tables; WITH (CTE) for temporary named subqueries.
    • In SELECT: scalar subqueries (must return a single value).

  • Data modification:

    • INSERT with VALUES or INSERT…SELECT; NULLs can be inserted explicitly.
    • DELETE with optional subqueries; beware changing aggregates during deletion (use CTEs to stage results).
    • UPDATE with SET and WHERE; use CASE for conditional updates; scalar subqueries can recompute derived columns.

  • Schema changes: DROP TABLE; ALTER TABLE ADD COLUMN (existing rows get NULL) and (limited) DROP COLUMN support.

  • Portability note: some/all set-comparison clauses may not be supported in all engines (e.g., SQLite).
