Entity Relationship (ER) Diagrams
2026-03-26 21:47   label UoA, Database

Concepts

Entity

Entity is a basic concept for the ER model. Entities are specific things or objects in the mini-world that are represented in the database.

Attribute

Attributes are properties used to describe an entity. A specific entity will have a value for each of its attributes. Each attribute has a value set (or data type) associated with it – e.g. integer, string, date, enumerated type, …

Types of Attributes

  • simple
    • Each entity has a single atomic value for the attribute
  • composite
    • The attribute may be composed of several components. For example:
      • Address(Apt#, House#, Street, City, State, ZipCode, Country),
      • Name(FirstName, MiddleName, LastName).
  • multi-valued
    • have multiple values for that attribute

Derived Attribute

whose value is dynamic and derived from another attribute.

Key Attribute

An attribute of an entity type for which each entity must have a unique value

  • A key attribute may be composite.
  • An entity type may have more than one key

Domains (Value Sets) of Attributes

A value set specifies the set of values associated with an attribute

Relationships

The degree of a relationship type is the number of participating entity types.

  • 1:1
  • N:1 or 1:N
  • M:N

ER notation

  • Entity (rectangle box 矩形)
    • Weak Entity (double retangle 双矩形)
  • Attribute (oval 椭圆)
    • Composite attribute (多个属性有下划线)
    • Multivalued attribute (double ovals 双椭圆)
    • Derived attribute (dash oval 虚线椭圆)
    • Key atttribute (underline 下划线)
      • partial key (虚线下划线)
  • Relationship (Diamond-shaped box 菱形)
    • not shown with an arrow
    • readable from left to right and top to bottom.
    • total（双线）必须参与
    • partial（单线）可以不参与
    • identifying relationship (双菱形, 依赖其他 entity 的主键)
